「Webコンテンツ作成」の成果

本テーマについて

  • 目的

    • 先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得すること。

  • 到達目標
    1. 手順書を読み、実験を行い、レポートやプレゼンテーションにより報告するという実験の一連の流れを実践できること。
    2. コミュニケーション能力の重要性を理解し、グループワークを円滑に実施できること。
    3. HTML等を用いたWebページ作成ができる。なお、本授業は、リーダーシップ能力、課題に対する解決能力育成に寄与する科目です。

      プロジェクト課題について

      今回のプロジェクトでは、グルメ情報を提供するWebページの作成に取り組みました。HTMLとCSSを使って見やすく操作しやすい画面を作り、JavaScriptで動きをつけています。特にAxiosというライブラリを使い、外部のAPIから飲食店の情報を非同期で取得できるようにしました。ユーザーが選んだジャンルに合わせて、関連するお店の情報を画面にリアルタイムで表示する機能を実装しています。これにより、利用者は簡単に好みのジャンルの飲食店を探せるようになり、快適に使えるサイトに仕上げることができました。

      プロジェクト課題のページ:プロジェクト課題のページ

      数当てゲーム

      この数当てゲームでは、1から10までの整数の中からランダムに選ばれた数を、ユーザーが予想して当てる仕組みになっています。テキストボックスに予想した数字を入力し、「回答」ボタンを押すと、JavaScriptが現在の予想回数をカウントしながら、入力された値と正解の数字を比較します。予想が当たっていれば正解と表示され、違っていればヒントとして「もっと大きいです」「もっと小さいです」といったメッセージが表示されます。また、何回目の予想かも画面上に表示されるようになっており、ゲーム性を高めています。HTMLで基本の構造を作り、JavaScriptで動作や判定、メッセージの表示処理を行っている点が特徴です。

      数当てゲームのページ:数当てゲームのページ

      各授業回の学習事項

      第1回 WWW と HTML

      今回の講義ではHTMLの基本的な知識、文章の構成とHTMLを使ってWebページを作成する方法を学びました。また、画像の添付方法、HTML要素の使い方、要素でハイパーリンクを設置する方法やWebページのデバッグの仕方を学びました。

      演習の成果:演習1-4: HTMLのさまざまな要素

      第2回 CSS による Web ページの装飾

      今回の講義ではCSSの基本的な知識、CSSを使ってWebページを装飾する方法を学びました。CSSのセレクタ、プロパティ、値の使い方、外部スタイルシートの作成方法、インラインスタイルと内部スタイルの違いについて学びました。

      演習の成果:演習2-3: ウェブブラウザ説明ページの装飾

      第3回 JavaScript の基本

      今回の講義ではJavaScriptの基本的な知識、JavaScriptを使ってWebページに動きを加える方法を学びました。コンソールを使ってエラーを探し、修正する方法を学びました。また、JavaとJavaScriptの二つの言語の構文の違いとJavaScriptのデバッグの仕方を学びました。

      演習の成果:オブジェクトの参照

      第4回 DOM と JavaScript

      今回の講義では、HTMLのinput要素の種類と属性、JavaScriptを使ったDOM操作の基本を学びました。querySelectorによる要素検索、textContentを使ったテキスト操作、addEventListenerを用いたイベント処理など、動的なWebページ作成の基礎を習得。特に、ボタンクリック時にフォームの入力値を取得して表示する実践的なインタラクションを作成することで、イベント駆動型プログラミングの重要な概念を理解しました。

      演習の成果:演習4-10: 足し算ページ

      第5回 イベント処理

      今回の講義では、JavaScriptを使ったDOM操作の基本を学びました。'createElement'で新しい要素を作成し、'insertAdjacentElement'を使ってリストに追加する方法、'setAttribute'で属性を変更する方法、'remove'で要素を削除する方法を実際に試しました。また、複数の要素を組み合わせて新しい構造を作り、ページに追加する練習も行いました。

      演習の成果:演習5-7: DOMの演習その2

      工夫点，苦心点，強調点，感想

      工夫点: 外部のAPIから飲食店の情報を取得し、ジャンルに応じた検索結果をリアルタイムで表示する機能を実装したことです。Axiosというライブラリを使用することで、非同期通信を行い、ユーザーが検索ボタンを押すと同時に画面が動的に変化するようにしました。選んだジャンルに応じた情報を素早く表示することで、利用者が使いやすいと感じられるような設計を意識しました。また、選択肢のジャンルを整理し、どのような料理ジャンルがあるかひと目で分かるようなUIに整えた点も工夫の一つです。

      苦心点: JavaScriptによるDOMの操作と、非同期処理の理解です。初めは、APIから取得したデータがどのような形式になっているのか把握することすら難しく、うまく画面に表示できずに何度もエラーを出してしまいました。コンソールを使って何度もエラーの内容を確認し、どこに問題があるのか地道に探しながら修正を繰り返しました。特に、DOMに新しい要素を追加する際の書き方や、データをループで処理して表示する方法に苦戦しました。

      強調点: Web制作においてHTML、CSS、JavaScriptの三つの技術が相互に連携していることです。単に見た目を整えるだけでなく、操作性や動作の滑らかさを実現するには、それぞれの役割を理解した上で、組み合わせる力が必要だと感じました。例えば、HTMLで適切な構造を作り、CSSで見やすくデザインし、その上でJavaScriptを用いて動きを加えることで、初めて「使えるページ」が完成しました。

      感想: 今回の課題を通して、最初はわからないことだらけで戸惑うことも多くありましたが、自分の手で調べながら一つずつ実装を進めていくうちに、できることが増えていく実感を得ることができました。完成したページを実際に動かしてみて、情報が正しく表示されたときには大きな達成感がありました。今後は、今回得た知識と経験をもとに、さらにユーザーにとって使いやすいデザインや機能を追求し、より完成度の高いWebページを作れるように技術を高めていきたいです。
